このプロジェクトで何をやっているのか

スマートフォン向けのお金管理アプリを、AIと一緒に設計・開発しています。
このガイドでは「何が作られていて」「どう進んでいるか」をわかりやすく説明します。

🗺 全体の流れ

このプロジェクトは大きく3つのものが組み合わさっています。

📱

資産管理アプリ

給与・カード・口座を管理するスマホ向けWebアプリ。実際に使えるものを作っています。

🤖

cc-sdd（開発の進め方）

AIに実装させる前に「要件→設計→タスク」を順番に固める仕組み。手戻りを減らします。

📊

ダッシュボード

開発の進捗をブラウザで確認・承認できるツール。このHTMLの隣で動いています。

📱 資産管理アプリについて

スマートフォン（iPhone / Android）で使えるWebアプリです。4つのタブで構成されています。

タブ 何ができるか 状態
① 給与シミュレーション 残業時間を入力すると手取り額をリアルタイム計算。固定項目は昇給時に編集可能 ✅ 完成
② クレジットカード管理 JCB・VISAの固定費・変動費を登録し、締め日ごとに集計 🔨 開発中
③ 銀行口座・証券口座 複数口座の残高をイベントで管理し、推移を見る 🔜 未着手
④ 過去データ 源泉徴収票・月別給与・残業時間のグラフ表示 🔜 未着手

アプリを起動するには

# ターミナルで asset-mgmt フォルダに移動してから実行 npm run dev

起動後、ブラウザで http://localhost:5173 を開くと表示されます。

💡 データはすべてブラウザ内に保存されます。インターネット接続不要・外部への送信ゼロです。

🤖 cc-sdd とは何か

AIにコードを書かせるとき、いきなり「作って」と言うと意図と違うものができることがあります。 cc-sdd は「先に設計書を固めてから実装する」というルールを守るための仕組みです。

なぜ使うのか

✋

手戻りを減らす

「こんなつもりじゃなかった」を防ぐために、要件と設計を先に人間が確認します。

📝

記録が残る

何を作ると決めたか・なぜそう設計したかが`.kiro/specs/`フォルダにファイルとして残ります。

🔄

段階的に進める

📊 進捗ダッシュボードについて

開発の進み具合をブラウザで確認・操作できるツールです。

起動方法

# ターミナルで asset-mgmt フォルダに移動してから実行 node dashboard.js

起動後、ブラウザで http://localhost:4000 を開きます。

画面の見方

1

ステッパー（横の進捗バー）

初期化 → 要件定義 → 設計 → タスク分解 → 実装 → 完了 の順番で進みます。現在地が濃い色で表示されます。

2

承認状況のチェックボックス

「生成済み」「承認」のチェックをONにすると、その場で保存されてフェーズが自動的に進みます。

3

ファイルプレビュー

requirements.md / design.md / tasks.md の内容を折りたたんで確認できます。クリックで開きます。

4

実装OKボタン

全フェーズが承認済みになったら「実装OK」にチェックを入れます。これでAIが実装を開始できる状態になります。

⚠️ アプリ（port 5173）とダッシュボード（port 4000）は別々のサーバーです。両方同時に起動できます。

🚀 今後の進め方

新しいタブを開発するときは、毎回この手順で進めます。

1

Claude Code に指示する

「/kiro:spec-requirements credit-card」のようにコマンドを入力すると、AIが要件定義の草案を作ります。

2

ダッシュボードで内容を確認する

http://localhost:4000 を開いて、生成された requirements.md を読みます。内容が合っていればチェックをONにします。

3

次のフェーズへ進む

承認したら Claude Code に「次へ」と伝えるだけ。設計・タスク分解も同じ手順で進めます。

4

実装OKを押す → 実装される

ダッシュボードで「実装OK」にチェックを入れ、Claude Code に「実装してください」と伝えると実装が始まります。

❓ よくある疑問

このアプリを使うのに料金はかかりますか？
アプリ自体・ダッシュボード・cc-sdd はすべて無料です。費用が発生するのは Claude Code（AI）の利用料のみで、Anthropic のプラン料金に含まれます。
入力したデータはどこに保存されますか？
すべてお使いのブラウザ内（localStorage）に保存されます。外部のサーバーには一切送信されません。ただしブラウザのデータ削除・別の端末では引き継がれません。
ダッシュボードで承認したのにAIが動かないのはなぜですか？
ダッシュボードはファイルを更新するだけです。承認後は Claude Code のチャット欄で「次のフェーズに進んでください」と入力してください。
port 5173 と port 4000 の違いは何ですか？
5173 は実際に動くアプリです（npm run dev で起動）。
4000 は開発の進捗を確認するダッシュボードです（node dashboard.js で起動）。
どちらも同時に起動できます。
.kiro フォルダや spec.json は何ですか？
cc-sdd が使う設定・仕様書のフォルダです。直接編集しなくても構いません。ダッシュボードの承認操作が spec.json を自動更新します。
アプリを他の人のスマホでも見られますか？
同じ Wi-Fi 内なら可能です。npm run dev -- --host で起動し、ターミナルに表示された「Network:」のURLを他の端末で開いてください。
